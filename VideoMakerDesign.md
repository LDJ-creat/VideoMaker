# VideoMaker初始设计方案

 
## 功能架构
 
### 用户交互
 
1. 用户输入爆款视频视频 → 结构拆解->沉淀为知识库
2. 用户输入视频制作要求（文字/图片/视频） →  内容分析 → 脚本生成->素材补全（图片生成，分镜视频等等）->视频组装->可编辑（自然语言调整或者时间轴式的剪辑）
→ 视频 片段组
3. 用户输入爆款视频视频+新视频制作要求→ 结构拆解 → 知识沉淀 → 内容分析 →  脚本生成->素材补全（图片生成，分镜视频等等）->视频组装（包含TTS,音频，字幕（可选）等等）->可编辑（自然语言调整或者时间轴式的剪辑）
 
 
 
### 核心模块拆解
 
### 1.知识沉淀：采用Karpathy提出的LLM wiki模式（https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f）而不是使用rag，以md文档或者数据库的形式存储，将知识库skill化，以skill 格式编写。
 
1.素材输入（本地或外部链接--外部链接使用yt-dlp进行下载）
 
提取（使用openCV,ffmpeg，fast-whisper和算法实现关键帧和文案的提取并调用LLM进行分析，必要时可以适当将其分辨率，音频的分析可能可以使用librosa / pydub）
 
- 整体画面、网标、短视频
- 文案：如何判断为BGM 还是口播？
- 音频与BGM的内容、节奏
- 元信息：时长、内容、分辨率…
 
2.结构拆解
 
- 脚本/段落结构：如开头 hook、中段展开、结尾 CTA
- 节奏结构：如镜头切换频率、段落快慢变化、高潮位置
- 包装结构：如字幕密度、标题条、贴纸、转场、封面风格等

3.知识沉淀和归档
将结构拆解结果转化为skill化的md
文档并进行归类，存储到知识库中。知识库中的归类可以按照（视频类别-风格这样的逻辑存储），如果已有相似的知识经验，LLM应该判断应该；作为新的风格进行新增还是在原有skill的基础上进行补充和优化

 
 
 
### 2.视频的生成
 
（多模态）
 
1. 对用户输入的内容的分析
用户的输入可能包含要使用的图片和视频素材以及文字版的视频要求，我们需要调用LLM对素材（多模态）进行分析然后选定对应的skill(如果没有找到合适的则置空)
 

 
 
2. 脚本生成
 
可选人机交互（通过开关进行控制，默认关闭）：
用户对脚本进行审核，并可以手动修改或通过自然语言提出建议让LLM进行修改
 
 
3.分镜拆分和素材补全
根据脚本拆分分镜，并评估用户是否提供了充足的脚本，如果缺乏则
 - 1.参考AI短剧的制作方式，通过提示词->调用生图模型生成关键图片-> 调用视频模型生成分镜视频补充素材。
 - 2.可以使用hyperframes进行生成
 选择哪一种方式有LLM根据所要生成的视频的种类，风格进行确定

可选人机交互（通过开关进行控制，默认关闭）：
用户对分镜内容和素材进行审核，并可以手动修改或通过自然语言提出建议让LLM进行修改（可以修改文本，视频或者视频）
 
4. 视频的内容组装与生成
重点使用hyperframes的能力实现
- TTS的合成，连贯性与一致性
- 背景音乐（可选）
- 转场
上述为我当前想到的内容，可能不完善需要补充

5.用户审核与调整
审核模式： 
 ① 自动化：依靠LLM进行评估和优化  
 ② 人工 + AI： 用户通过自然语言提出改进建议，AI根据建议进行优化
 ③ 纯人工：像剪辑软件一样可以通过时间轴剪辑  

6.复盘与进化
在完成最终视频制作后，根据整个制作链路种用户提出的建议和用户的反馈进行复盘，并沉淀为经验，让用户审核是否需要修改，是否认同，如果用户同意后则更新到知识库中。
其中的用户审核为可选人机交互（通过开关设置，默认开启）

### 工程化
1.开发过程中的可观测性：通过langfuse和关键日志，关键产物的落盘（落盘到文件中）进行监测和评估（每次任务单独记录）
2.中断恢复：当出现断网或者服务异常时（无论是视频结构拆解与知识沉淀还是视频的生成，在前端提供友好的提示和重试按钮，在进行重试时可以复用此前已有的产物继续任务，无需从头开始）


### 用户体验
1.知识库内容在前端的可视化展示
2.生成视频的持久话粗出以及多版本管理--对于已生成的视频，用户可以通过自然语言进行修改生成新版本（不覆盖旧版本）
3.生成的视频的分类与搜索功能的支持
 

## 技术架构
前端： Next.js

后端核心：python

数据库：还没有想好使用sqlite还是使用postgre,同时对于沉淀的知识库，我还没有想好是存到数据库中还是直接在特定目录下使用文件存储。

模型：文本模型，生图模型，生视频模型,TTS,音乐生成（可能需要）都调用外部API

其他依赖：fast-whisper, ffmpeg,openCV,librosa / pydub， hyperframes, langfuse
